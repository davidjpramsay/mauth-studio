Human review surface. Agent control surface.

Mauth keeps assessment authoring local. The web app remains the place where a teacher reviews layout, diagrams, solutions, files, and print output. The bridge gives local agents a structured contract for making changes.

snapshot -> dry run -> apply -> validate -> browser check

For teachers

Build printable high-school mathematics tests and worksheets with questions, parts, subparts, diagrams, tables, working space, and solutions.

  • Preview first: A4 pages render in the browser before print.
  • Recover safely: Autosave, project files, logos, and versions stay local.
  • Review clearly: Agent comments and suggestions are separate from committed edits.

For Codex and Claude

Work through explicit document state and deterministic actions instead of guessing React state or editing saved project JSON directly.

  • Read: `mauth_snapshot` exposes stable ids and validation state.
  • Preview: `mauth_actions_preview` dry-runs action batches.
  • Apply: `mauth_actions_apply` commits through history, autosave, and revision checks.

Alpha software

Mauth Studio is still in active development. Expect app code, schemas, docs, tests, and agent workflows to change as the local bridge matures.

  • Fork or clone: fork the repo for your own public copy, or clone it locally for an agent-operated project.
  • Change the app: agents can edit source code, tests, docs, and CI in a normal Git workflow.
  • Keep user data out: do not commit local `storage/`, drafts, temporary output, or generated files.

Two chat work streams

Keep development and assessment authoring in separate agent chats where practical.

  • Mauth Development: app code, architecture, schemas, tests, docs, CI, and repository maintenance.
  • Mauth Authoring: create, inspect, convert, and polish assessments through the app and bridge.
  • Shared contract: both streams should read `AGENTS.md` and the bridge/action docs.

Local-first, MCP-ready.

The HTTP bridge is the canonical contract. MCP wraps that bridge for Claude Desktop, Codex, and other local tools. Successful edits still flow through the same web app action layer the human UI uses.
