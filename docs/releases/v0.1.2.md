# Mauth Studio 0.1.2

Mauth Studio 0.1.2 is an Apple Silicon alpha release focused on self-contained agent setup, structured assessment authoring, and safer document handling.

## Download

- macOS on Apple Silicon (`arm64`)
- Developer ID signed and notarized by Apple
- Distributed as `Mauth-Studio-0.1.2-arm64.dmg`

Open the DMG, move **Mauth Studio** to Applications, then launch it from Finder, Spotlight, or the Dock. Python, Node.js, a repository checkout, and open Terminal windows are not required for teacher use or the bundled agent connector.

Existing updater-enabled builds can install 0.1.2 through **Mauth Studio > Check for Updates...** after approving the download and restart. A direct DMG install remains available.

## Included

- A self-contained Mauth Agent Connector inside the signed app bundle
- One-time Codex, Claude Code, and Claude Desktop setup details under **Help > Set Up Codex or Claude...**
- Runtime connector verification without persisting the private per-launch bridge token
- Standard tests with multiple full title pages and **T** title-page controls
- Investigation assessments with a shared student brief and teacher rubric
- Canonical `.mauth` document files and Finder open-document integration
- Direct question, part, and subpart wording editing
- Expression-aware diagram numeric fields such as `pi/2`, `sqrt(2)`, and `1/3`
- Connected graph angle markers, independent axis-end arrows, and draggable axis labels
- Existing tests, exams, worksheets, notes, diagrams, solutions, storage, backup, print, and update workflows

## Agent Setup

1. Keep Mauth Studio in Applications and open it.
2. Choose **Help > Set Up Codex or Claude...**.
3. Copy and run the command for Codex or Claude Code, or merge the shown `mauth` entry into Claude Desktop's Developer configuration.
4. Keep Mauth Studio open while the agent uses its Mauth tools.

The connector discovers the current dynamic loopback address and random token from Mauth's private mode-`0600` runtime manifest. The token is never copied into the agent configuration. A repository checkout is required only for app development.

## Release Safety

- Mauth does not download or install updates without teacher confirmation.
- The updater is enabled only in packaged releases with signed update metadata.
- Development builds do not check for updates.
- Packaging verification starts the bundled connector with the app-owned Electron runtime.
- Publication remains draft-first until the DMG, ZIP, updater metadata, and blockmaps pass remote verification.

## Alpha Limitations

- Apple Silicon only; Intel and universal builds are not currently supplied.
- This is early software. Use **Backup ZIP** regularly and keep important assessment files in a backed-up folder.
- Codex, Claude Code, or Claude Desktop must be installed separately if that agent integration is wanted.

## Development

Use `pnpm macos:dev` for source iteration. Use `pnpm smoke:agent-connector` to verify MCP negotiation against a running development app. `pnpm macos:release` and `pnpm macos:ship` remain reserved for traceable external releases.
