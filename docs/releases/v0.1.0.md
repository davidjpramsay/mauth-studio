# Mauth Studio 0.1.0

Mauth Studio 0.1.0 is the first public alpha of the standalone macOS app for authoring printable mathematics assessments.

## Download

- macOS on Apple Silicon (`arm64`)
- Developer ID signed and notarized by Apple
- Distributed as `Mauth Studio-0.1.0-arm64.dmg`

Open the DMG, move **Mauth Studio** to Applications, then launch it from Finder, Spotlight, or the Dock. Python, Node.js, a repository checkout, and open Terminal windows are not required for normal use.

## Included

- School test, exam booklet, worksheet, and mathematics-notes templates
- Questions, parts, subparts, choices, tables, working spaces, and page controls
- Student and manually editable Solutions modes
- MathJax mathematics plus JSXGraph, Penrose, Plotly, vector, 3D, network, Venn, and image diagram surfaces
- Visible local document files, autosave recovery, versions, reusable logos, external-folder selection, and ZIP backup/import
- Optional authenticated local bridge for Codex, Claude Code, Cursor, and MCP clients
- Blank new tests, exams, and worksheets that remain blank after reopening

## Verification

- Full repository quality gate passed: formatting, lint, API tests, web/action tests, Plotly tests, launcher/runtime tests, TypeScript, and the production build
- App and DMG passed Developer ID signing, Apple notarization, ticket stapling, and Gatekeeper validation
- Installed-app checks covered dynamic sidecar startup, authenticated agent discovery, document creation, autosave recovery, and clean quit behavior

## Alpha Limitations

- Apple Silicon only; Intel and universal builds are not currently supplied
- Updates are manual: download and install a newer signed DMG when another release is published
- This is early software. Use **Backup ZIP** regularly and keep important assessment files in a backed-up folder
- A separate clean-Mac download/install verification remains part of the release-hardening queue
- Codex/Claude bridge helper commands currently require a local repository checkout and Node dependencies; the standalone app itself does not

## Development

Use `pnpm macos:dev` for source iteration. `pnpm macos:release` is reserved for versioned external releases and should not run after every source change.
