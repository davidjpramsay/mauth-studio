# Development

Purpose: build and maintain the Mauth app.

Start here for app code, architecture, schemas, docs, tests, repository maintenance, CI, and release work.

Required context:

1. Read `AGENTS.md`.
2. Read `docs/current-state.md`.
3. Read `docs/local-ai-workflow.md`.
4. Read `docs/agent-bridge.md` when changing authoring automation, action contracts, project-file safety, validation, or external-agent surfaces.
5. Read the specific docs/tests for the subsystem being changed.
6. Keep generated artifacts in `workspace/`.
7. Run targeted tests before handing work back.

Default posture: make durable app improvements through source, tests, docs, and structured contracts.
